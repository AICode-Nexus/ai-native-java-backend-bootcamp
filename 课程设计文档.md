# 《AI-Native Java Backend Bootcamp》课程设计文档

## 一、课程定位

核心命题：

**不是教“Java 如何调用一个模型接口”，而是教“如何用 Java / Spring Boot 构建可上线的 AI-Native 后端系统”。**

### 目标受众

- 主受众：AI 应用开发者
- 次受众：希望补足后端和全栈能力的前端工程师

### 教学主线

以“企业知识助手”为主业务场景，把模型接入、RAG、Agent、Workflow、工程治理和生产化串成一条完整链路。

## 二、教学方法

### 1. 深讲一个，横向对比多个

- 深讲：Java / Spring Boot 主线
- 对比：Go、Python、TypeScript 的最小对照样例

### 2. 业务链路优先

课程以业务能力组织，而不是以中间件清单组织。

### 3. 工程治理贯穿

从课程中期开始，持续引入评测、安全、日志、成本与交付意识。

## 三、课程结构

| 课程 | 核心目标 | 主项目增量 |
|------|----------|------------|
| 第0课 | 建立 AI-Native 后端认知 | 明确架构分层 |
| 第1课 | 接入模型与服务层 | `ai/model` |
| 第2课 | 聊天接口与流式输出 | `chat` |
| 第3课 | Prompt 与结构化输出 | `ai/prompt` `ai/parser` |
| 第4课 | 文件解析与切片 | `knowledge/document` |
| 第5课 | RAG 链路 | `knowledge/rag` |
| 第6课 | 检索协同 | `search` `vector` `cache` |
| 第7课 | Tool Calling | `agent/tool` |
| 第8课 | Workflow 与异步任务 | `workflow` `task` |
| 第9课 | 工程化治理 | `ops` `ai/safety` `evals` |
| 第10课 | 业务融合 | `dataassistant` |
| 第11课 | 全链路整合 | 生产化与交付 |

## 四、工程主项目

`demo/` 采用“教学友好的模块化单体”：

- Java 21
- Spring Boot 3.x
- Spring AI + LangChain4j
- MySQL + Redis + Elasticsearch + RabbitMQ
- 向量检索作为可替换边界层

## 五、课程资产

- 每课一个 `final-content.md`
- 每课一组课后练习
- 一个贯穿主项目
- 一组跨语言对比样例
- 一组架构图与流程图
