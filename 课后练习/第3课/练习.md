## 基础题

### 题目

给问答接口增加一层 Prompt 模板封装，不要把完整 Prompt 字符串散落在 Controller 里。

### 建议路径

- 先识别哪些内容是固定模板
- 再把变量占位抽出来
- 让业务输入和 Prompt 参数形成稳定映射

### 交付物

- 一段 Prompt 模板封装代码
- 一份参数说明

## 进阶题

### 题目

加入一层上下文记忆结构，让多轮问题至少能带上最近一轮上下文。

### 建议路径

- 先明确上下文放在什么层
- 限制上下文长度，不要无限追加
- 说明上下文如何参与模型输入

### 交付物

- 一个上下文结构定义
- 一条多轮对话示例

## 挑战题

### 题目

把回答改成结构化响应对象，至少显式区分主回答、模式或额外说明字段。

### 建议路径

- 先定义 DTO，再反推接口字段
- 不要把所有结果塞进一段自然语言

### 交付物

- 一个结构化响应对象
- 至少一条接口或服务测试

## 验收标准

- Prompt 参数可复用
- 上下文结构可解释
- 输出字段完整且有明确职责
