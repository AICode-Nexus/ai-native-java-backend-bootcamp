# 课后练习

每节主线课都配套三档练习：

1. 基础题：跑通本课核心工作流
2. 进阶题：把 AI 协作结果落成可验证产物
3. 挑战题：加入边界、风险或团队协作约束

所有练习都优先基于 `demo/` 示例项目或仓库现有文档完成，不建议另起一个完全脱离课程语境的新项目。

## 怎么使用这套练习

推荐顺序如下：

1. 先读对应课程的 `final-content.md`
2. 明确本课要优化的研发动作
3. 对照 `demo/` 或顶层文档收集上下文
4. 先产出结构化方案，再让 AI 辅助生成细节
5. 最后通过测试、构建、日志、文档审查或自检清单验证结果

## 建议交付方式

每道练习建议至少交付下面几类产物中的两类：

- 结构化需求或设计说明
- 接口契约或数据结构草案
- 代码增量或重构 diff
- 测试用例、样例或回归清单
- 发布检查单、复盘记录或团队规范草案

## 统一验收原则

无论哪一课，练习结果都建议同时满足：

- 能说清边界，而不是只堆生成内容
- 能说明 AI 在哪个环节起作用
- 能被测试、构建、日志或文档审查验证
- 能沉淀为以后仍可复用的模板或流程
