# 课后练习

每节课都配套三档练习：

1. 基础题：跑通课堂主能力
2. 进阶题：做一个受控扩展
3. 挑战题：引入治理、性能或业务约束

所有练习都优先基于 `demo/` 主项目增量完成，不建议另起一个完全脱离课程主线的新项目。

## 怎么使用这套练习

推荐顺序如下：

1. 先读对应课程的 `final-content.md`
2. 再运行一次 `demo/` 现有测试
3. 只做当前课程对应的最小增量
4. 优先补测试或验收脚本，再补实现
5. 最后对照“验收标准”自检

## 建议交付方式

每道练习建议至少交付下面几类产物中的两类：

- 代码增量
- 测试用例
- 一段简短设计说明
- 一份接口或流程示例输入输出

## 统一验收原则

无论哪一课，练习结果都建议同时满足：

- 能说清边界，而不是只堆功能
- 能说明为什么这样设计
- 能被代码、测试或文档验证
- 能和课程主项目主线对齐
