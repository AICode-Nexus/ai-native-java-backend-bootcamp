# 第0课 练习

## 练习定位

这一课不是写复杂代码，而是建立整套训练营的系统地图。  
你的目标是把“AI-Native Java 后端到底由哪些层组成”讲清楚。

## 参考位置

- 课程正文：[`../../第0课-认知重构/final-content.md`](../../第0课-认知重构/final-content.md)
- 课程总览：[`../../README.md`](../../README.md)
- 课程设计文档：[`../../课程设计文档.md`](../../课程设计文档.md)

## 基础题

### 题目

画出当前训练营主项目的五层结构图：API、AI Service、Retrieval / Tool / Workflow、Data / Infra、Governance。

### 建议路径

- 先从课程正文里整理层次
- 再对照 `README.md` 和 `课程设计文档.md`
- 每一层只写 2 到 4 个关键职责，不要堆概念

### 交付物

- 一张结构图，或一份分层清单
- 每层一句职责说明

## 进阶题

### 题目

用 300 到 500 字说明“传统 CRUD 后端”和“AI-Native 后端”的核心差异。

### 建议路径

- 从输入、处理中间层、成功标准三个角度展开
- 不要写成抽象口号，尽量落到当前课程主线

### 交付物

- 一段结构化说明文字

## 挑战题

### 题目

为你自己的一个业务系统，写一版 AI-Native 改造草图，说明你会先改哪一条链路、为什么。

### 建议路径

- 先选一个小场景，例如知识问答、数据查询或文档理解
- 明确边界，不要一开始就设计成“万能 Agent”

### 交付物

- 一页改造草图或一份简短设计说明

## 验收标准

- 能清楚说出系统分层
- 能解释 AI-Native 与传统后端的差别
- 选择的改造切入点具体、可执行
