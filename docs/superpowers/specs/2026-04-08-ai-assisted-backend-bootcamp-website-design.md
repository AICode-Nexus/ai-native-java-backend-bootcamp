# AI 辅助后端研发 Bootcamp 课程网站设计文档

## 背景

仓库已经从“AI-Native Java Backend 系统建设课程”改轨为“传统 Java 后端开发如何使用 AI 提效研发全流程”的主线课程。

这次改轨带来两个直接结果：

1. 课程主线不再围绕模型接入、RAG、Tools、Workflow 与治理生产化逐课展开
2. 原有 AI-Native 内容仍然保留，但应降级为独立进阶专题，而不是继续占据主线课位

因此，课程网站也必须同步改轨，不能继续把旧主线当成唯一入口。

## 目标

新的课程网站需要完成四件事：

1. 首页默认传达“AI 辅助后端研发”的课程定位
2. `/learn` 仅承载 12 节主线课程
3. `/advanced` 单独承载 4 个进阶专题
4. 搜索、导航与阅读路径显式区分“主线课程”和“进阶专题”

## 非目标

以下内容不属于这次设计改造范围：

1. 不把课程网站与 `demo/` 运行时做耦合
2. 不引入账号、评论、CMS 等站点能力
3. 不把所有正文改为 MDX 或数据库驱动
4. 不把网站目标扩展成营销站或社区平台

## 核心决策

### 1. 双区结构取代单主线结构

网站不再只服务一个课程目录，而改为：

- 主线课程：AI 辅助后端研发全生命周期
- 进阶专题：AI-Native 系统建设相关内容

### 2. 根目录 Markdown 仍是唯一正文来源

主线课程和进阶专题都继续使用仓库根目录下的 `final-content.md` 作为正文唯一来源，站点只维护展示层元数据。

### 3. `demo/` 的站内语义改变

`demo/` 不再被解释为“课程最终要搭出的 AI 产品”，而是：

- 真实 Java / Spring Boot 示例项目
- 给需求理解、设计、测试、排障、重构和文档协作提供素材

## 信息架构

### 站点级路由

改轨后的核心路由为：

1. `/`
   课程首页，强调“AI 辅助后端研发”定位
2. `/learn`
   主线课程总览
3. `/learn/[lessonId]`
   单节主线课程页
4. `/advanced`
   进阶专题总览
5. `/advanced/[topicId]`
   单个进阶专题页

### 首页 `/`

首页应包含：

1. Hero 区
   明确课程主定位是“传统后端研发如何使用 AI 提效”
2. 学习路径区
   清楚区分主线课程与进阶专题
3. 生命周期提效地图
   用需求、设计、编码、测试、排障、发布、团队落地组织内容
4. 主线课程预览
5. 进阶专题预览
6. 核心原则与资源入口

### 主线课程区 `/learn`

这一页只回答：

- 主线课程教什么
- 12 节课分别覆盖哪个研发动作
- 学习时应先看哪条路径

### 进阶专题区 `/advanced`

这一页只回答：

- 什么内容属于主线之外的进阶阅读
- 原有 AI-Native 内容如何作为系统建设专题继续存在

## 数据模型

站点元数据从单一 `lessons` 扩展为两套集合：

### `mainCourses`

包含：

- `id`
- `number`
- `title`
- `subtitle`
- `duration`
- `tags`
- `dirName`
- `summary`
- `section = "main"`
- `sectionLabel = "主线课程"`
- `href`

### `advancedTopics`

包含与主线相同的展示字段，但 `section = "advanced"`，`sectionLabel = "进阶专题"`。

### 搜索模型

搜索索引不再只收录主线课程，而是统一收录主线课程与进阶专题。  
搜索结果需显式展示 `sectionLabel`，避免用户误把专题当成主线课。

## 导航与交互

### 顶部 Header

保留全站统一 Header，但导航项从旧版的“网站首页 / 课程学习”扩展为：

- 网站首页
- 主线课程
- 进阶专题
- GitHub

### 左侧 Sidebar

侧边导航必须分两组展示：

- 主线课程
- 进阶专题

并且支持：

- `/learn` 和 `/advanced` 总览入口
- 当前页高亮
- 移动端抽屉模式

### 搜索 Dialog

搜索结果需同时支持：

- 主线课程命中
- 进阶专题命中
- 标题、副标题、标签、正文片段命中

并在结果卡片里显示：

- `sectionLabel`
- 课程 / 专题编号
- 命中类型

## 验收标准

这次设计是否落地，以以下标准判断：

1. 首页第一屏不再默认传达“AI-Native 后端系统建设”定位
2. `/learn` 只列主线课程，`/advanced` 只列进阶专题
3. 搜索能同时命中主线课程与进阶专题，并显示分类
4. 导航和侧边栏能显式区分主线课程与进阶专题
5. 网站内容模型与仓库中的课程目录、专题目录保持一致
